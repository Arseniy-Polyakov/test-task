Тестовое задание
Контекст
Мы — инвестиционная компания, предоставляющая клиентам финансовые услуги.
Помимо инвестиционных продуктов, у компании есть база образовательных курсов по финансовой тематике.
В компании также реализована рекомендательная система, которая формирует для клиентов различные персональные рекомендации, в том числе курсы.
Рекомендации формируются еженедельно.
Для каждого клиента доступен список рекомендованных курсов со следующими полями:
•	recommended_course
•	course_description
•	probability — вероятность релевантности (0–100)
•	price
•	discount — скидка на текущую неделю
________________________________________
Задача
Разработать прототип диалогового ассистента для персонализированных рекомендаций курсов, использующего методы машинного обучения и/или современные LLM-подходы.
Ассистент должен:
•	принимать user_id (идентификатор клиента),
•	работать с его weekly-рекомендациями,
•	вести персонализированный диалог,
•	предлагать релевантные курсы с учётом контекста общения.
Архитектурные решения и выбор технологий остаются на усмотрение кандидата.
________________________________________
Что должно быть реализовано
1. Интерфейс
•	Страница с полем для ввода user_id
•	Кнопка «Начать диалог»
•	Окно чата:
o	история сообщений
o	поле для ввода вопроса
•	Ассистент отвечает в режиме диалога
•	Диалог персонализирован под введённый user_id
________________________________________
2. Логика ассистента
•	Ассистент получает список weekly-рекомендаций клиента (файл df.csv)
•	Предлагает курсы с учётом:
o	probability
o	price
o	discount
•	Если пользователь ничего не уточняет — предлагает наиболее релевантные варианты
•	Если пользователь пишет, например:
o	«хочу курс по акциям»
o	«дорого»
o	«что попроще?»
ассистент должен учитывать запрос и предлагать релевантные варианты из доступного списка
•	Ассистент должен учитывать контекст диалога
________________________________________
3. ML / AI-составляющая
•	Использование probability в логике ранжирования
•	Обработка пользовательских намерений (любым способом)
•	Допускается использование LLM
•	Допускается использование embeddings / семантического поиска
•	Обоснование выбранного подхода должно быть отражено в README
________________________________________

•	Хранение контекста диалога
•	Структурированный код
